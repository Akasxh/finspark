# AdaptConfig — Claude Code Project Guide

## What this is

AI-powered integration configuration platform for Indian fintech lending. Upload API specs (CIBIL, eKYC, GST, UPI, Paytm, Razorpay), auto-generate field mappings via LLM, simulate integrations, manage lifecycle (Draft -> Active), and inspect API security — all through a web UI, REST API, or MCP server.

**Team:** Nucleolus (S Akash, Swayam Jain, Yash Kamdar) — IIT Patna

## Quick start

```bash
# Backend
uv sync --extra dev
cp .env.example .env   # then fill in API keys
uv run uvicorn finspark.main:app --reload --port 8000

# Frontend
cd frontend && npm install && npm run dev   # localhost:5173

# Tests
uv run python -m pytest tests/ --no-cov     # 1297 tests

# MCP server (for Claude Desktop / IDE agents)
uv run adaptconfig-mcp
```

```
  schemas/           # Pydantic request/response models
  services/
    parsing/         # Document parser (LLM + regex)
    config_engine/   # Field mapper, validator, diff engine, rollback

    lint/            # Spectral OpenAPI linter
    security/        # OWASP API Top 10 security inspector
    llm/             # LLM clients (OpenAI GPT-5, OpenRouter Claude, Gemini)
  mcp/               # MCP server (6 tools for LLM-driven automation)
  seeds/             # Adapter catalog JSON + admin user seeder

frontend/src/
  pages/             # Dashboard, Documents, Configurations, Adapters, Simulations, Webhooks, Search, Audit
  lib/               # API client (axios), auth utils
  components/        # Toast, shared UI
  types/             # TypeScript interfaces
```

## LLM provider

The app uses LLMs for 5 tasks: document parsing, config generation, field mapping, simulation validation, search query understanding. Provider is configurable:

| Provider | Env var | Default model | Reasoning model |
|----------|---------|---------------|-----------------|
| **OpenAI** (default) | `FINSPARK_OPENAI_API_KEY` | `gpt-5-mini` | `gpt-5` |
| OpenRouter | `FINSPARK_OPENROUTER_API_KEY` | `anthropic/claude-sonnet-4-6` | `anthropic/claude-opus-4-7` |
| Gemini | `FINSPARK_GEMINI_API_KEY` | `gemini-3-flash-preview` | — |

Switch with `FINSPARK_LLM_PROVIDER=openai|openrouter|gemini`. All call sites use `get_llm_client()` — same interface regardless of provider.

**GPT-5 quirk:** Reasoning models only accept default temperature (1.0). The `OpenAIClient` auto-detects reasoning models and drops the temperature param.

## Key features

- **API chaining:** Endpoints have `id`, `depends_on`, `extract` (JSONPath), `inject` (template strings). Supports OAuth -> resource -> status flows.
- **Spectral linting:** Auto-runs on uploaded OpenAPI/AsyncAPI specs. POST `/api/v1/lint/` for ad-hoc linting.
- **Security inspector:** 10 OWASP API Top 10 rule-based checks + LLM semantic analysis. POST `/api/v1/security/inspect-spec`.
- **MCP server:** `adaptconfig-mcp` exposes parse/generate/simulate/search/list tools over stdio.
- **Webhook delivery:** Fires on real events (document.uploaded, config.created, simulation.completed, etc.) via FastAPI BackgroundTasks.

## Coding conventions

- **Python:** strict types, `ruff` for linting, `uv` for deps, `pytest` for tests.
- **Frontend:** React 18, TypeScript strict, Tailwind, Recharts.
- **Tests:** `uv run python -m pytest tests/` (NOT `uv run pytest` — different sys.path behavior). Use `--no-cov` for speed.
- **Commits:** conventional commits: `feat(scope): description`, `fix(scope): description`.
- **Git identity:** Always use `Akasxh <drakathakash@gmail.com>`. The `git-identity.sh` hook sometimes resets to `Akasxh-autostep` — override with `git config user.name Akasxh && git config user.email drakathakash@gmail.com` if needed.
- **No AI traces:** Never include "Generated by Claude" or similar in code, commits, PRs, or issues.
- **Functions:** under 50 lines. Files: under 300 lines (soft).
- **Error handling:** mandatory — no bare except/catch.
- **No hardcoded secrets** — use `FINSPARK_*` env vars via `settings`.

## Coordination rules (multi-developer)

When multiple people work on this repo simultaneously:

1. **Check `.claude/coordination/TASKS.md` first** — if it exists, a multi-session collaboration is active. Read it to see who owns what files.
2. **Claim your work** — before starting, add your task to TASKS.md with your name and the files you'll touch.
3. **Never edit files someone else claimed** — if you need to modify a claimed file, coordinate first.
4. **Use feature branches** — `feature/your-name/description`, not direct-to-main.
5. **Log activity** to `.claude/coordination/LOG.md` when starting/finishing work.
6. **Check open GitHub issues** before starting — `gh issue list --state open`. If an issue is assigned, don't duplicate the work.
7. **Assign yourself** to the GitHub issue you're working on.

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `FINSPARK_LLM_PROVIDER` | Yes | `openai` / `openrouter` / `gemini` |
| `FINSPARK_OPENAI_API_KEY` | If openai | OpenAI API key (sk-proj-...) |
| `FINSPARK_OPENROUTER_API_KEY` | If openrouter | OpenRouter key (sk-or-v1-...) |
| `FINSPARK_GEMINI_API_KEY` | If gemini | Google AI key |
| `FINSPARK_LLM_MODEL` | No | Default model (e.g. `gpt-5-mini`) |
| `FINSPARK_LLM_MODEL_REASONING` | No | Reasoning model (e.g. `gpt-5`) |
| `FINSPARK_AI_ENABLED` | Yes | `true` to enable LLM features |
| `FINSPARK_SECRET_KEY` | Prod | JWT signing key (32+ chars) |
| `FINSPARK_ENCRYPTION_KEY` | Prod | Webhook secret encryption (32+ chars) |
| `FINSPARK_DATABASE_URL` | No | Default: `sqlite+aiosqlite:///./adaptconfig.db` |
| `FINSPARK_ADMIN_PASSWORD` | Prod | Admin user password (fails fast if missing in prod) |
| `FINSPARK_DEBUG` | No | `true` for dev mode |
